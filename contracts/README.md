# Governance Contract

This directory defines the governance-to-implementation contract consumed by `context-engineering-implementation`.

## Canonical Contract

- `governance-implementation-contract.json`

## Versioning Policy

- Semantic versioning is required.
- `MAJOR`: breaking interface/contract changes.
- `MINOR`: backward-compatible additive changes.
- `PATCH`: non-interface corrections.

## Change Control

- Contract changes require Compliance Officer review.
- Breaking changes require coordinated implementation updates before merge.
